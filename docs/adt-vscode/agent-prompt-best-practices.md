# Best Practices for Agent Prompts

Effective agent prompts are essential for successful AI-driven development workflows. Well-structured prompts enable agents to understand requirements, follow proper procedures, and deliver quality results.

The following recommendations constitute best practices for agent prompting:

Give complete context- Provide the same details you would give a colleague.

Break tasks into steps - Use numbered steps (1, 2, 3…) to help the agent stay focused.

Treat it like pair programming - Work together as you would with a human teammate.

Stop and redirect when needed - If the agent goes off track, intervene with better instructions.

Don’t let it struggle - Provide information directly instead of watching it search.

Use examples - Attach sample code to help the agent understand faster.

Add relevant packages - Include necessary APIs in the explorer view so the agent can find them.

Keep context focused - Too much information can reduce the agent’s effectiveness.
